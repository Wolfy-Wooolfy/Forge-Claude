## §4 Key Technical Decisions

- **SPA fallback scope:** `GET && !req.url.startsWith("/api/")` — catches React Router paths (`/chat`, `/projects`, `/vision`, `/kb`, `/doctor`) without intercepting API routes. `/assets/*` handler placed before the fallback.
- **Self-cleaning build:** `clean.js` deletes only `../../index.html` and `../../assets/` — never `web/server.js` or `web/index.legacy.html`. Wired as `"prebuild"` so stale hashed chunks cannot accumulate.
- **API-base behaviour change (recorded, not fixed):** `web/server.js` still injects `window.__FORGE_API_BASE__` on `GET /`, but the React app reads `import.meta.env.VITE_API_BASE` (baked at build time). The legacy runtime `FORGE_API_PORT` override no longer applies to the frontend. Standard setup (port 3100) unaffected. Runtime port configurability — if needed — requires a build-time mechanism. Out of PHASE-13 scope.
- **Accessibility:** All 5 fixes are colour/label changes only — no structural change to the UI.
